# 書籍作成シミュレーター用 紙データベース設計

印刷営業が図録・写真集・絵本・文芸書などの紙選びを支援するためのツール向け設計。
ページ数・判型・製本方式を入力すると、本文重量・背幅・紙の質感・向き不向き・
営業説明文まで出せることをゴールとする。最初は**本文紙**に絞る。

関連ファイル:
- `schema.sql` … SQLite向けDDL（本設計の2をそのまま実行できる形にしたもの）
- `index.html` / `css/style.css` / `js/*.js` … 4章の計算式(本文用紙重量・背幅)を
  実際に試せる最小プロトタイプ。紙・判型・製本方式のデータは`js/data.js`の
  架空サンプル値で、実在銘柄のスペックではない。

### プロトタイプの動かし方・テスト方法

このリポジトリには他アプリと同様ビルド手順はない。ブラウザで直接確認する。

1. リポジトリ直下で `python3 -m http.server` を実行し、
   `http://localhost:8000/paper-database/index.html` を開く
   （もしくは `paper-database/index.html` を直接ブラウザで開いてもよい）。
2. デフォルト値（200ページ・A5判・書籍用紙A・並製本）のまま「計算する」を押すと、
   本節の本文用紙重量の計算例（≒253g）と一致することを確認できる。
3. 計算ロジック(`js/calc.js`)はDOMに依存しない純粋関数なので、Node単体でも検証できる:
   `node paper-database/js/calc.test.js`（design.mdに書いた4つの計算例をアサートする）。

## 1. 必要なデータ項目一覧

### 1-1. 銘柄基本情報
- メーカー名 / メーカーコード
- シリーズ名（例: b7バルキー、OKトップコート+ など）
- 銘柄名・商品名
- 商品コード（メーカー品番）
- 紙の分類（上質紙／書籍用紙／嵩高書籍用紙／コート紙／マットコート紙／微塗工紙 等）
- 発売年、廃盤フラグ
- FSC/PEFC等の環境認証、再生紙比率
- 取扱商社、参考価格、最小ロット、標準納期

### 1-2. 物理特性（数値スペック）
- 米坪（g/m²）… 銘柄は複数米坪を展開するため「銘柄×米坪」が1行の粒度
- 連量（kg）… 基準原紙寸法（四六判 or 菊判）とセットで管理
- 紙厚（μm）
- 白色度（%）
- 光沢度（%、グロス値）
- 不透明度（%）
- （参考指標として）比容積・かさ密度 cm³/g … 嵩高紙の判定に有用

### 1-3. 質感評価（5段階、社内基準で統一）
- ラフ感
- 硬さ（コシ）
- 嵩高感
- しっとり感
- 高級感
- 読みやすさ
- 写真再現性

### 1-4. データ出所の区分（必須：公式値と実測値を混同しない）
- `official`（メーカーカタログ・技術資料からの転記）— 出典URL/PDF名、転記日
- `measured`（自社で実測）— 測定者、測定日、測定方法、サンプルロット、信頼度（1-5）
- `sales_note`（営業の所感・案件メモ）— 自由記述、担当者、案件名、日付、タグ

### 1-5. 用途適性
- 用途ジャンル（図録／写真集／絵本／文芸書 等、コンテンツ軸）
- 製本方式（上製本／並製本、構造軸。ジャンルと直交する別軸として管理）
- 適性スコア（1-5）＋ 理由テキスト（営業説明文の元ネタになる）

### 1-6. シミュレーション用の書籍側入力
- ページ数
- 判型（仕上がり寸法 mm）
- 製本方式（背幅補正・重量補正のルールに紐づく）
- 選定した本文紙銘柄・米坪
- （任意）表紙紙・付属物

## 2. テーブル設計

ER関係の要点:
- `papers`（銘柄マスタ）を中心に、`paper_official_specs` / `paper_measured_specs` は
  **1銘柄に対して複数行**（米坪違い・測定履歴違い）を許容する1:N。
- `paper_texture_ratings` と `paper_sales_notes` も同様に1:N。出所が
  official/measured/sales_note のどれかを常にタグ付けする。
- 用途適性は `paper_usage_fitness` が `papers` と `book_genres` の中間テーブル（多対多）。
- シミュレーターは `book_projects`（入力）→ `book_calculation_results`（計算結果）で分離し、
  同じ入力に対する再計算履歴を残せるようにする。

```
manufacturers ─┬─< papers >─┬─< paper_official_specs
               │            ├─< paper_measured_specs
               │            ├─< paper_texture_ratings
               │            ├─< paper_sales_notes
               │            └─< paper_usage_fitness >─ book_genres
               │
trim_sizes ────┼─< book_projects >── binding_types
               │        │
papers ────────┘        └─< book_calculation_results
```

詳細な列定義は `schema.sql` を参照（SQLite DDLとしてそのまま実行可能）。
主要テーブルの要約:

| テーブル | 粒度 | 主な列 |
|---|---|---|
| `manufacturers` | メーカー1社 | 名称、公式サイト |
| `papers` | 銘柄1つ | メーカーFK、銘柄名、分類、廃盤フラグ、認証 |
| `paper_official_specs` | 銘柄×米坪×基準寸法 | 米坪、連量、紙厚、白色度、光沢度、不透明度、出典 |
| `paper_measured_specs` | 銘柄×測定回 | 上記数値＋測定者/日/方法/信頼度 |
| `paper_texture_ratings` | 銘柄×評価回×出所 | 7項目の5段階評価 |
| `paper_sales_notes` | 銘柄×メモ1件 | 自由記述、案件名、タグ |
| `book_genres` | ジャンル1つ | 図録／写真集／絵本／文芸書 等 |
| `paper_usage_fitness` | 銘柄×ジャンル | 適性スコア、理由 |
| `trim_sizes` | 判型1つ | 仕上がり幅高さ mm |
| `binding_types` | 製本方式1つ | 背幅補正式のパラメータ |
| `book_projects` | シミュレーション入力1件 | ページ数、判型FK、製本FK、紙FK |
| `book_calculation_results` | 計算結果1件 | 重量、背幅、おすすめ判定、営業文 |

## 3. Googleスプレッドシートで始める場合の列名

まずはシートを分けて運用し、後でCSV→DBに移行できるよう列名をテーブル設計に合わせる。

### シート「紙マスタ_公式」
| メーカー名 | シリーズ名 | 銘柄名 | 商品コード | 分類 | 基準寸法(四六判/菊判) | 米坪(g/m2) | 連量(kg) | 紙厚(um) | 白色度(%) | 光沢度(%) | 不透明度(%) | 出典URL | 出典資料名 | 転記日 | 廃盤 | FSC等認証 |

### シート「紙マスタ_実測」
| 銘柄名 | 米坪(g/m2) | 紙厚(um) | 白色度(%) | 光沢度(%) | 不透明度(%) | 測定者 | 測定日 | 測定方法 | サンプルロット | 信頼度(1-5) | 備考 |

### シート「質感評価」
| 銘柄名 | 出所(official/measured/sales_note) | ラフ感 | 硬さ | 嵩高感 | しっとり感 | 高級感 | 読みやすさ | 写真再現性 | 評価者 | 評価日 | 備考 |

### シート「営業メモ」
| 銘柄名 | 日付 | 担当者 | 案件名 | メモ内容 | タグ(カンマ区切り) | 総合評価(1-5) |

### シート「用途適性」
| 銘柄名 | 用途ジャンル(図録/写真集/絵本/文芸書) | 適性スコア(1-5) | 理由 |

### シート「判型マスタ」
| 判型名 | 仕上がり幅(mm) | 仕上がり高さ(mm) |

### シート「製本方式マスタ」
| 製本方式名 | 背幅補正係数 | 見返し等の固定加算(mm) | 備考 |

### シート「シミュレーション入力」
| プロジェクト名 | ページ数 | 判型名 | 製本方式名 | 本文紙銘柄 | 本文紙米坪 | 作成日 | 作成者 |

列名を最初からDB設計と1:1にしておくことで、後のCSVインポート/SQLite移行がスクリプト1本で済む。

## 4. 重量・背幅の計算式

### 前提知識（原紙基準寸法）
- 四六判原紙: 788mm × 1091mm → 面積 0.85958 m²
- 菊判原紙: 636mm × 939mm → 面積 0.59720 m²

### 米坪 ⇔ 連量の換算
```
米坪(g/m²) = 連量(kg) ÷ 原紙1枚の面積(m²)
```
例: 四六判・連量70kgの場合 → 70 ÷ 0.85958 ≒ 81.4 g/m²

### 紙厚の目安（かさ高紙の判定に使用）
```
紙厚(mm) ≒ 米坪(g/m²) × 比容積(cm³/g) ÷ 1000
```
比容積（1/かさ密度）が大きいほど「同じ米坪でも厚くて軽い」＝嵩高紙。
一般の書籍用紙は比容積 1.0〜1.3 cm³/g程度、嵩高書籍用紙は 1.3〜1.8 cm³/g程度が目安
（銘柄ごとの実測値を `paper_measured_specs` に登録して置き換える）。

### 本文用紙重量
```
枚数(葉) = ページ数 ÷ 2　※奇数ページは切り上げ
仕上がり面積(m²) = 仕上がり幅(mm) × 仕上がり高さ(mm) ÷ 1,000,000
本文用紙重量(g) = 枚数 × 仕上がり面積(m²) × 米坪(g/m²)
```
例: 200ページ・A5判(148×210mm)・米坪81.4g/m²
→ 枚数100、面積0.03108 m² → 100 × 0.03108 × 81.4 ≒ 253g

### 背幅
```
背幅(mm) = 紙厚(mm) × 枚数 × 背幅補正係数 + 製本方式の固定加算(mm)
```
- 並製本（無線綴じ）: 補正係数 ≒ 1.00〜1.05（のりの入り込み分）、固定加算はほぼ0
- 上製本（丸背・角背）: 補正係数 ≒ 1.00、固定加算に見返し（遊び紙）2枚分の厚み・
  花布/寒冷紗の余裕を数mm加算（`binding_types.固定加算(mm)` に銘柄非依存の値として保持）

これらの補正係数・固定加算は実測ベースで営業メモから随時チューニングする前提。

### 書籍総重量（概算）
```
総重量(g) = 本文用紙重量 + 表紙重量 + 付属物重量(任意: オビ・スリーブ等)
表紙重量(並製本, g) = 表紙紙米坪(g/m²) × 表紙仕上がり面積(m²) × 表紙枚数(通常1)
表紙重量(上製本, g) = 芯ボール紙重量 + 表紙材(クロス/紙)重量  ※上製本は別途固定値テーブルで管理
```

## 5. 最初に登録すべき紙銘柄カテゴリ

本文紙を「読み物中心」「写真・図版中心」「絵本・実用書」の3軸で最初に揃える。
下記は代表的な銘柄"カテゴリ"の例。実際の米坪表・連量表・厚みは登録時に
必ずメーカー公式カタログ（PDF）から `paper_official_specs` へ転記し、
このドキュメントの数値を鵜呑みにしないこと。

| カテゴリ | 用途 | 特徴 | 代表的な銘柄例（要カタログ確認） |
|---|---|---|---|
| 書籍用紙（クリーム系） | 文芸書・小説 | 目に優しい色味、めくりやすさ重視 | 淡クリームキンマリ、オペラクリーム系 |
| 嵩高書籍用紙 | 文芸書の軽量化・厚み演出 | 低米坪でも厚みが出て軽い | b7バルキー、b7トラネクスト系 |
| 上質紙 | 並製本一般・実用書 | 汎用、コストバランス型 | 各社の上質紙70kg/90kg |
| マットコート紙 | 絵本・実用書・図録の一部 | 適度な光沢とインク乗り | 各社マットコート系 |
| 微塗工紙 | 絵本・児童書 | 発色と柔らかい質感の両立 | 各社微塗工紙 |
| コート紙・アート紙 | 写真集・図録 | 高光沢・高発色・写真再現性重視 | 各社アート紙・コート紙 |

登録の優先順位はこの表の上から（＝文芸書向け）ではなく、営業が今扱っている案件の
ジャンルに合わせてよい。ただし「1カテゴリにつき最低1銘柄」は初期投入時点で揃えると
用途別おすすめ判定が最初から機能する。

## 6. MVPとして作るべき画面構成

1. **紙マスタ一覧・検索画面**
   分類・米坪レンジ・用途ジャンル・質感（例:写真再現性4以上）で絞り込み検索。
2. **紙詳細画面**
   公式値／実測値／営業メモをタブで分離表示。質感7項目はレーダーチャートで可視化。
3. **シミュレーター入力画面**
   ページ数・判型・製本方式・本文紙銘柄（米坪）を選択。
4. **シミュレーション結果画面**
   本文重量・背幅・書籍総重量（概算）・価格概算・その紙の用途適性コメント・
   同ジャンル向けの代替候補紙リスト・営業説明文の自動生成（テンプレ＋質感評価から文章化）。
5. **紙比較画面**
   2〜4銘柄を横並びで数値・質感レーダー・適性スコアを比較。
6. **データ管理画面**
   実測値・営業メモの入力/編集フォーム、CSV/JSONのインポート・エクスポート。

MVPの実装順は 1→3→4→2→6→5 が現実的（検索とシミュレーション計算が核、
比較画面は後回しでよい）。

## 7. 将来的な拡張案

- 表紙紙・特殊紙（箔押し・エンボス・PP加工等）を別テーブルとして追加し、
  表紙側の重量・コストもシミュレーションに含める。
- 印刷コスト概算（紙単価・断裁ロス率・製本費）との連携、見積もり自動生成。
- 印刷機／製本ラインごとの適性（オフセット輪転機、デジタル印刷機との相性）。
- 環境認証・CO2排出量など、SDGs観点のデータ項目追加。
- 「この紙を使った実例書籍」の紐付け（実績ギャラリー、営業提案時の参考事例）。
- 質感評価と用途適性の相関から、AIによる自動レコメンド（現状は人手でスコア入力）。
- 海外メーカー紙・多言語対応（インバウンド出版案件向け）。
- サンプル/断裁見本の在庫管理・発注連携。
